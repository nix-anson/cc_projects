Generate a Django model with common fields and best practices.

Arguments:
- $1: App name where the model will be created (required)
- $2: Model name in PascalCase (required)

Steps to execute:

1. Verify the app exists

2. Ask the user what fields the model should have (or use common defaults):
   - Primary key (auto-generated by Django)
   - Common fields (name, description, etc.)
   - Relationships (ForeignKey, ManyToMany)
   - Timestamps (created_at, updated_at)

3. Generate the model code following Django best practices:
   - Use appropriate field types
   - Add related_name for relationships
   - Include __str__ method
   - Add Meta class with ordering and verbose names
   - Add indexes for frequently queried fields
   - Include docstrings

4. Add the model to the app's models.py file

5. Register the model in admin.py with basic ModelAdmin

6. Remind to run:
   - `python manage.py makemigrations`
   - `python manage.py migrate`

Example model template:
```python
class $2(models.Model):
    """$2 model description."""

    # Fields
    name = models.CharField(max_length=200)
    slug = models.SlugField(unique=True)
    description = models.TextField(blank=True)

    # Timestamps
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        ordering = ['-created_at']
        verbose_name = '$2'
        verbose_name_plural = '${2}s'
        indexes = [
            models.Index(fields=['slug']),
        ]

    def __str__(self):
        return self.name
```

Best practices:
- Use singular names for models
- Always define __str__ method
- Add appropriate indexes
- Use related_name for relationships
- Include created_at/updated_at timestamps
